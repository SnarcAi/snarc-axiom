# Contributing

Thanks for your interest in SNARC AXIOM.

## What we accept
- clarifications to the paper/spec
- tightening proofs and definitions
- threat-model improvements
- test vectors / consistency checks
- documentation improvements

## What we do NOT accept
- token launches, exchange listings, fundraising
- claims of “guaranteed security” or “unbreakable”
- production custody or “real value” deployment guides

## How to contribute
1. Open an Issue describing the change.
2. Submit a PR referencing the Issue.
3. Keep changes focused and well-scoped.

## Style
- Prefer precise language.
- When introducing assumptions, state them explicitly.
- Keep the repo research-only.
